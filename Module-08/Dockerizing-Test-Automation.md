# Dockerizing Test Automation for Scalable Test Execution

## Project Objectives

In this project, you are to containerize a functional test suite using Docker. Also, you will learn to isolate test environments, remove dependency conflicts, and improve test execution consistency across platforms.

---

## Project Description

Understand the purpose and benefits of Docker in QA workflows  

Learn how to write a Dockerfile to containerize automation projects  

Gain hands-on experience with building, running, and debugging test containers  

Integrate containerized tests into CI pipelines  

---

## Tasks

A working test suite  

UI tests for Swag Labs (Login, cart and checkout) with Selenium WebDriver.  

Dockerize the test environment:  

Write a Dockerfile to build a container image  

Include required libraries and dependencies in the Docker image  

Set up entry commands to execute the test suite automatically when the container runs  

Build and run the container:  

Use docker build and docker run to validate everything runs correctly in the container  

Automate the build/test with GitHub Actions:  

Set up a simple CI pipeline to trigger the test suite in a container
